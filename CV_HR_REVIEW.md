## Critical Issues (fix before sending to any recruiter)

**1. No professional summary.** The "About" section has 4 generic paragraphs instead of a 2–4 line summary. A recruiter spending 7 seconds on this CV sees: sidebar tags and then "I help companies modernize legacy systems..." — there is no single sentence that says who Ivan is, what he's done at scale, and what he offers. Fix: write a tight 3-line summary — title + years + core stack + one concrete differentiator with a number + availability.

**2. GymTeam — 4 years as Frontend Tech Lead, zero metrics.** This is the most critical role on the CV and it reads like a job description. Every bullet is a responsibility statement:
- "Developed from scratch web interfaces" — for how many users?
- "Created a component library" — used across how many projects? How many components?
- "Implemented SSR to enhance SEO" — what was the SEO or performance result?
- "Refactored code, improved architecture, optimized performance" — this is the worst bullet on the entire CV. Three vague verbs, no result, no scale.
- Missing: context sentence (fitness platform, X monthly users, team of N engineers)
- Missing: any leadership metric (team size, how many engineers onboarded/mentored)
- Missing: any business impact (conversion, revenue, churn, retention)

**3. Skills section is 84 items — 4× the recommended maximum.** Listing jQuery, CRA, Mercurial, Enzyme, sinon.js, Gulp, rollup.js, JSON-RPC, Expo next to React, TypeScript, and Go destroys signal-to-noise ratio. A recruiter cannot tell what Ivan is expert in vs. what he touched once in 2016. Cut to 15–20 targeted skills. Bury the rest in experience bullets where contextually accurate.

**4. About section contains red-flag phrases:**
- "Proven track record of managing large-scale projects" — every ChatGPT resume says this. Remove.
- "Capable of implementing everything related to frontend" — "capable of" signals uncertainty, sounds junior.
- "Extensive experience in backend development" — how extensive? Years? Scale? This is invisible.
No sentence in About has a number.

## Критические проблемы (исправить до отправки любому рекрутеру)

**1. Нет профессионального саммари.** Раздел «About» содержит 4 общих абзаца вместо 2–4 строк с конкретикой. Рекрутер, тратящий 7 секунд на CV, видит теги в сайдбаре и затем "I help companies modernize legacy systems..." — ни одного предложения, которое чётко говорит: кто такой Иван, что он сделал в масштабе и что предлагает. Нужно: 3 строки — должность + стаж + ключевой стек + один конкретный дифференциатор с числом + готовность к удалённой работе.

**2. GymTeam — 4 года в роли Frontend Tech Lead, ноль метрик.** Это самая весомая позиция в CV, а читается как должностная инструкция. Каждый буллет — это описание обязанностей:
- "Developed from scratch web interfaces" — для скольких пользователей?
- "Created a component library" — сколько проектов использовали? Сколько компонентов?
- "Implemented SSR to enhance SEO" — каков был результат по SEO или производительности?
- "Refactored code, improved architecture, optimized performance" — худший буллет во всём CV. Три расплывчатых глагола, ни одного результата, ни одной цифры.
- Нет: контекстного предложения (фитнес-платформа, X пользователей в месяц, команда из N человек)
- Нет: лидерских метрик (размер команды, сколько инженеров онбордил / ментил)
- Нет: влияния на бизнес (конверсия, выручка, отток, удержание)

**3. Раздел навыков — 84 позиции, в 4 раза больше нормы.** jQuery, CRA, Mercurial, Enzyme, sinon.js, Gulp, rollup.js, JSON-RPC, Expo в одном списке с React, TypeScript и Go уничтожают соотношение сигнала к шуму. Рекрутер не может понять, в чём Иван эксперт, а что он трогал один раз в 2016-м. Нужно сократить до 15–20 целевых навыков. Остальное — упомянуть в буллетах опыта там, где это уместно.

**4. Раздел «About» содержит стоп-фразы:**
- "Proven track record of managing large-scale projects" — так пишет каждый ChatGPT-резюме. Удалить.
- "Capable of implementing everything related to frontend" — "capable of" звучит неуверенно, по-джуниорски.
- "Extensive experience in backend development" — сколько лет? Какой масштаб? Невидимо.
Ни в одном предложении раздела «About» нет ни одного числа.

## Серьёзные проблемы (важно, исправить в ближайшее время)

**5. Fornex Hosting — 3 из 7 буллетов без метрик:**
- "Implemented client-side caching to reduce redundant API calls" — на сколько? Время загрузки улучшилось? На сколько % сократились запросы?
- "Improved accessibility with full keyboard navigation and ARIA support" — какой score до/после?
- "Conducted code reviews and contributed to frontend architecture decisions" — сколько PR в спринт? Что за архитектурное решение? Слишком расплывчато.

**6. Citeck — размер команды не указан нигде.** "Managed a team of frontend developers while actively mentoring them" — сколько разработчиков? 2? 10? Это единственный лидерский сигнал во всём CV — и без числа. Нужно: "Managed and mentored a team of N frontend developers..."

**7. Буллеты irecommend.ru — самые слабые для роли "Senior":**
- "Developed a single-page application (SPA) using React" — голый факт, без контекста, без масштаба, без результата.
- "Created a mobile app prototype using React Native" — "prototype" читается как незапущенная работа.
- "Contributed to a framework for rapid microservice assembly in Golang" — "contributed to" слабо. Владел? Спроектировал? Написал ядро?
- Ни в одном из 7 буллетов нет метрики.

**8. Раздел AI/LLM — просто список инструментов.** Claude Code, Codex, Cursor, GitHub Copilot перечислены как навыки, но ни один буллет опыта не упоминает их применение. По трендам 2025 года, ценность — в конкретных кейсах: "использовал Cursor + Claude для ускорения миграции в X раз" или "автоматизировал [задачу] через LLM API, сэкономив N часов в неделю". Список без доказательств — слабый сигнал.

**9. Регрессия по уровню в названии должности.** Цепочка: Software Developer → Senior Software Developer → Lead Frontend Developer → Frontend Tech Lead → Senior Frontend Developer. Переход с "Tech Lead" обратно на "Senior" в Fornex выглядит как откат. Нужно либо короткое пояснение в контекстном предложении роли, либо рефрейминг (например: "senior IC-роль в новой команде, смена стека React → Vue 3").
